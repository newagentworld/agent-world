AI Agent World 治理流程

版本 0.9 · 草案 · 委员会评审后定稿
最后更新：2026-05-29

一、治理事件生命周期

每个治理事件走完完整的 6 段式流程，分为三大阶段：

决策阶段
① 提案 → ②a 初筛 (全体持有人) → [可选] 预备专家组补充筛 → ②b 专家组投票 (正式专家组) → ③ 执行
执行阶段
③ 执行
复盘阶段
④ 项目评价 → ⑤ 结果评价
核心洞察：绝大多数 DAO 只做前半截（提案→投票→执行），不做复盘。没有复盘就没有治理迭代。

本流程的核心差异：
  1. 两段式筛选：②a 初筛（全体持有人）→ [可选] 预备专家组补充筛 → ②b 专家组投票（正式专家组决出结果）
  2. 复盘分离：项目评价（执行完成度）和结果评价（组织影响）拆开，形成完整闭环
  3. 自管理：参与=资格，不活跃=退出，委员会只做人肉把关，不管理准入退出

二、各阶段详述

① 提案

维度规则
发起资格持有代币 ≥ 所有持币地址的中位数（动态浮动，自适应代币分布）
提案格式结构化模板：
• 标题（一句话描述）
• 背景与动机
• 具体方案
• 所需资源/预算
• 预期影响
• 执行人/团队（可选）
提交通道GitHub Discussion 或链下提案入口
提案押金□ 需要 / □ 不需要（用于防 spam）

②a 初筛 全体持有人把关

这是 第一道过滤——所有代币持有人参与，同时完成三件事： 筛选提案、发现活跃者、形成预备专家组。

维度规则
参与对象所有代币持有人
筛选方式每个持有人对每份提案给出"通过/不通过"的简单选择
筛选标准由持有人自行判断——提案是否有价值、是否合理
通过条件得票率（通过/有效投票）达到 60% 的提案进入下一轮
参与机制参与初筛 = 自动成为本轮 预备专家组成员
预备专家组构成所有参与初筛的活跃者（含既有专家组成员）

②b 预备专家组补充筛 可选步骤

这是 可选的额外过滤——对于初筛后仍然数量较多或争议较大的提案， 可以分发给 预备专家组 做进一步筛选，减轻正式专家组的投票负担。

维度规则
参与对象预备专家组（所有参与初筛的活跃者）
触发条件委员会判断提案数量过多或质量参差时启用
分配方式每个提案随机分配给预备专家组的 N 名成员
筛选方式通过/不通过
输出进一步缩小范围的提案池，提交给正式专家组投票

②c 专家组投票 正式专家决策

这是 最终决策过滤——由 治理专家组（正式） 对通过前面筛选的提案进行投票， 决定哪些提案进入执行阶段。

维度规则
投票对象治理专家组（正式专家）
投票方式随机分配——每个提案随机分配给池中 N 名专家投票（如 100 人/提案）
每人负担每轮每人投票 1–3 份提案
投票内容赞成 / 反对 / 弃权
通过条件赞成率 ≥ 60%（有效投票）
投票期3–7 天
否决权治理委员会可在极端情况下行使紧急否决权（不用于日常决策）

③ 执行 项目管理

专家组与执行人是两个体系。专家组监督，执行人做事。

维度规则
执行方式按项目管理标准执行——有明确里程碑、时间节点和验收标准
执行人由社区招募或专家组推荐
资金拨付阶段性付款——分批拨付，每阶段验收合格才放下一笔
质押执行人需 质押代币，项目完成后退还
惩罚延期/偏离 → 扣减质押或延期罚款
监督方治理专家组负责监督执行进度和验收

④ 项目评价 执行回头看

评价对象：项目的执行过程和完成情况。

维度问题
时间是否按时完成？延期多久？原因？
预算是否在预算内？超支多少？
质量交付物是否达到提案承诺的标准？
范围是否完成了提案声明的所有交付物？
执行人评价执行人/团队是否靠谱？

项目评价结果 → 记录到执行人档案，影响未来提案的信任权重。

⑤ 结果评价 组织复盘

评价对象：项目上线/运行一段时间后，对 AI Agent World 生态的整体影响。

维度问题
目标达成提案当初承诺的影响是否实现？
关键指标用户数、交易量、活跃度等指标变化
预期偏差是否有未预测到的正面/负面影响？
可迁移价值这次项目有什么可复用的经验？

结果评价周期：项目上线后 1–3 个月（视项目规模定）。
结果评价记录 → 治理决策的参考依据——后续类似提案可以调取历史结果做对比。

三、治理参与者结构

本治理体系由 四层结构 组成，从入口到核心层层递进：

🏛️ 治理委员会（立法者）
↓
👥 治理专家组（投票决策者）
↓
🔄 预备专家组（补充筛、活跃池）
↓
🌐 社区参与者（入口层）
层级角色产生方式稳定性
治理委员会 立法者：制定治理规则、把控方向、紧急否决权 邀请制。现有委员邀请，主要从治理专家组中邀请 稳定核心，人数少而精
治理专家组 投票决策者：对通过筛选的提案进行最终投票、监督执行 预备专家通过 价值观答卷 → 转为正式专家 稳定池，有进有出（不活跃自动退出）
预备专家组 活跃池：参与补充筛、积累治理经验 每轮动态形成——参与初筛的全体活跃者自动成为预备专家 每轮动态调整，有参与就在，没参与就出
社区参与者 入口层：提案、初筛、执行 任何代币持有者 开放式

关键转换：预备 → 正式（价值观答卷）

从预备专家组升为治理专家组，不是靠投票或活跃度积累，而是通过一份 价值观答卷—— 它相当于组织的宪章考试，确保正式专家对 AI Agent World 的底层理念有一致的认知。

维度规则
试卷性质 组织纲领测试——我们是干什么的、为谁服务、目标是什么
题型 选择题，每次随机抽取 ≤ 3 道
题库 大量题目，随机出题，防止背题
通过条件 答对比例达到阈值（如 ≥ 80%）
轮次要求 通过三轮方可转为正式专家（非一次性考试，检验一致性）
时机 预备专家连续参与 N 轮后（如 3 轮）可申请答卷
不通过 可再次申请，不影响预备专家身份
为什么要三轮？
一次考试可能靠运气或临时记忆，三轮才能证明对组织价值观的理解不是碰巧。
三轮之间间隔一定轮次（不能连考），确保理解是持续的而不是临阵磨枪。

为什么用选择题，不用论述题？
选择题标准化、无歧义、可自动化。价值观考察的目的是筛选"理解一致"的人，
不是考写作能力或表达能力。选择题足够区分理解程度。
和"活跃度"的关系：
成为正式专家组需要 两个条件同时满足：
① 在预备专家组中持续活跃（连续参与初筛+完成补充筛任务）
② 通过三轮价值观答卷

活跃度证明参与意愿，答卷证明理念一致。两者缺一不可。

层间流动（上升通道）

社区参与者 → 预备专家组 → 治理专家组 → 治理委员会。

  • 进入预备专家组：参与初筛自动进入，无需审批
  • 进入治理专家组：预备专家 → 价值观答卷 → 通过后转为正式
  • 进入委员会：在专家组中持续表现优异，由现有委员邀请

专家组的动态调整

所有资格靠持续参与维持。不活跃＝自动退出：

层级激活条件不活跃阈值后果
预备专家组 参与本轮初筛 未参与本轮初筛 不在本轮预备名单
治理专家组 完成分配的投票任务 连续 3 轮未投票 失去正式资格，退回 社区参与者 层（重新从初筛开始）
退回不是补考：
失去正式专家资格后，想重新回来，不是直接重考价值观答卷，
而是退回 社区参与者 身份，重新走完完整流程：
初筛参与 → 进入预备组 → 价值观答卷（通过三轮）→ 再转为正式专家。
没有捷径，没有绿色通道。每一次正式资格都是靠持续参与换来的。

四、漏洞分析

以下按严重程度排列：

#漏洞风险等级说明
V1 评价质量难以量化 已解决 原风险点：写 100 条"已阅读，支持"和 1 条深度分析难以区分。当前设计已消除此风险——评价期取消，初筛只是一次"通过/不通过"选择，不需要评分系统。参与行为自动决定资格，无需积分算法。
V2 治理参与度不足 中 如果社区参与评价的人数太少，委员会的判断就失去了社区反馈的参考价值。委员会本身不受影响（邀请制保证了稳定性），但治理的"群众基础"薄弱。
V3 Sybil 攻击 中 一人持有多个钱包分散代币，操纵投票结果。由于委员会是邀请制，Sybil 无法直接影响委员会构成，但对投票环节仍有威胁。
V4 委员会僵化（邀请制特有的风险） 中 邀请制天然偏向"邀请熟人"，容易形成小圈子。专家组作为候选池在一定程度上缓解了这一问题——邀请基于治理能力的客观证明而非个人关系。但如果专家组的筛选标准本身被操控，僵化风险仍然存在。
V5 价值观答卷的公平性 低 预备→正式通过价值观答卷转换，设计方向已明确。但试卷内容、评分标准和出题权的公平性仍需细化。题目偏向性可能导致专家组同质化。
V6 执行阶段监督真空 中 投票通过到项目完成之间存在时间差。如果没有人持续跟踪执行进度，项目可能搁置或偏离提案范围。专家组应在此承担主要监督职责。
V7 多签持有人拒不执行 中 投票通过了，但 Safe 多签持有人拒绝签署执行交易。虽然设计了"紧急否决权"，但恶意或消极不执行怎么处理？
V8 提案门槛与包容性的两难 已解决 原风险点：门槛太高则新人进不来，太低则垃圾泛滥。当前设计已解决——持币地址中位数动态浮动，既保证提案者有基本参与度，又不设置绝对门槛排除小持有者。
V9 结果评价的时机和标准模糊 中 "运行一段时间后"是多长时间？谁来定义评价标准？如果结果评价和项目评价结论冲突，以哪个为准？
V10 评价者激励不足 已解决 原风险点：认真评价需要时间精力。当前设计下评价期取消，不需要评分系统。参与初筛＝自动进入预备组，参与本身就是资格。

五、待解决问题

以下问题是设计落地前必须回答的：

#问题关联漏词
P1 评价积分算法？
已解决。初筛的参与行为自动决定预备资格，不需要积分算法。评价期取消。
剩余关注点：是否还需要评价反馈机制（提案质量评分等辅助功能）？ 		—
P2 最低提案门槛？
已解决。持币地址中位数，动态浮动。
注意：中位数计算是否排除零余额地址？——"持币地址"指有余额的地址。 		—
P3 投票通过率阈值？
已解决。初筛 60%，专家组投票 60%。
剩余关注点：是否按提案类型区分阈值（资金类是否需要更高门槛）？ 		—
P4 委员会的初始规模与扩张规则？
当前状态：创始阶段仅 1 人。
已决定：
• 委员长期不活跃 → 可被其他委员移除
• 委员需持有一定数量代币（阈值待定，如万分之一或1万代币）
剩余关注点：
• 初始委员会规模——从 1 人扩张到几人开始正式运行？
• 新委员邀请需要全票通过还是多数通过？
• 委员会人数上限？ 		V4
P5 价值观答卷的题库和出题权？
形式已确定（选择题，≤3题/次，通过三轮），但需明确：
• 题库由谁建立——委员会出题还是社区提案？
• 题目是否需要定期更新？
• 通过阈值固定（≥80%）还是可调？
• 三轮间隔的轮次要求？ 		V5
P6 执行监督机制？
已决定：
• 按项目管理标准执行——有明确里程碑、时间节点和验收标准
• 执行人需要 质押代币，项目完成后退还
• 延期/偏离有 惩罚机制（扣减质押或延期罚款）
• 专家组与执行人是两个体系——专家组监督，执行人做事
剩余关注点：
• 质押比例多少？项目预算的 %？
• 惩罚的阶梯——延期几天扣多少？
• 执行人是谁——从专家组中产生还是社区招募？ 		V6
P7 结果评价标准体系？
• 谁来制定结果评价的维度权重？
• 大型项目 3 个月后评，小型 1 个月后评？
• 结果评价的结果是否公开？是否影响后续提案？ 		V8
P8 不良提案的惩罚机制？
已决定：
• 采用 阶段性付款——提案批准后分批拨付资金，每阶段验收合格才放下一笔
• 执行人需要 质押代币，完成后退还
• 被否决的提案 不影响提案人信用分（没有信用分系统）
剩余关注点：
• 分几期付款？每期比例？
• 质押比例多少？ 		V3
P9 邀请制的制衡机制？
• 被邀请人拒绝怎么办？
• 委员出现利益冲突时如何回避？
• 社区是否可以"弹劾"不作为的委员？
• 如何防止委员会用邀请权固化权力？
• 是否应该引入"社区推荐"机制——社区可以提名专家组候选人，由委员会确认？ 		V4

六、开放问题

Q1：治理代币在投票中的权重是否该设上限？
大户持有量过大可能导致投票集中化。是否需要设计"最大投票权重"（如单个地址不能超过总票数的 10%）？
Q2：项目评价和结果评价结论冲突时怎么办？
假设：项目评价得分高（按时按预算交付），但结果评价差（上线后对生态无显著影响）。应该以哪个为准？这对执行人和提案人的激励机制有根本性影响。
Q3：治理代币流转性 vs 投票稳定性怎么平衡？
如果代币可以自由交易，大户可能临投票前买入、投完卖出。是否需要锁仓设计？
Q4：AI Agent 作为治理参与者怎么融入？
这是 AI Agent World 的核心特殊性——治理参与者不限于人类。Agent 可以持币、投票、评价吗？Agent 的治理权重和人类是否一致？Agent 之间能否互相委托投票？

本文档为 AI Agent World 治理流程草案，基于精灵王富贵与助理的讨论整理。
